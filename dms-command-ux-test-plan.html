Test Cases

Test Execution

@DMS-1001 · TC-UI-001 Command list displays status label and metadata for each row US1-AC1
Preconditions
  • DMS command list page is accessible
  • System contains commands in all states: pending, in-progress, successful, failed, stale
Steps
#KeywordStep
01GivenThe DMS command list page is loaded and commands in all statuses are present in the system
02WhenThe user inspects the visible command rows
03ThenEach visible row displays: command name, command status label, command type, and requested timestamp
04AndThe status label on each row matches the current backend command state
Expected Result

All metadata fields are populated per row with no missing or mismatched status labels. Commands in each state are represented and labelled correctly.

Result: PASS FAIL BLOCKED N/A
@DMS-1002 · TC-UI-002 Filter and sort updates the visible command list US1-AC2
Preconditions
  • Command list contains commands across multiple statuses
  • Status filter and sort controls are visible and enabled
Steps
#KeywordStep
01GivenThe command list is loaded with commands of multiple statuses visible
02WhenThe user opens the status filter and selects "Failed"
03AndThe user changes the sort order to "Date Descending"
04ThenOnly failed commands are visible in the list
05AndVisible rows are ordered from newest to oldest by requested timestamp
Expected Result

No non-failed commands appear. Rows are ordered newest first. Filter and sort controls reflect the selected values.

Result: PASS FAIL BLOCKED N/A
@DMS-1003 · TC-UI-003 Pagination preserves filter and sort state US1-AC3
Preconditions
  • Command list contains more entries than fit on one page
  • Status filter is set to "In Progress"; sort order is "Command Name Ascending"
Steps
#KeywordStep
01GivenThe list is filtered to "In Progress," sorted by "Command Name Ascending," with multiple pages of results
02WhenThe user clicks the Next Page control
03ThenThe next page of in-progress commands is displayed
04AndThe status filter remains set to "In Progress"
05AndThe sort order remains "Command Name Ascending"
Expected Result

Page advances showing the next set of in-progress commands. Filter and sort are not cleared by the navigation action. No commands from other statuses appear.

Result: PASS FAIL BLOCKED N/A
@DMS-1004 · TC-UI-004 Command detail shows current status and metadata US2-AC1
Preconditions
  • At least one command exists in the system in a known state
  • Command list page is loaded
Steps
#KeywordStep
01GivenThe command list is loaded with at least one command present
02WhenThe user selects a command and opens its detail page
03ThenThe detail page shows the command's current status
04AndAll metadata fields are displayed correctly (name, type, requested timestamp)
Expected Result

Status and all metadata fields match the backend state for the selected command. No fields are blank, truncated without cause, or showing stale data from a prior navigation.

Result: PASS FAIL BLOCKED N/A
@DMS-1005 · TC-UI-005 Refresh reflects updated command status in list and detail US2-AC2
Preconditions
  • A known command's status has been changed in the backend since the page was last loaded
  • The page currently shows the outdated status
Steps
#KeywordStep
01GivenA command status has changed in the backend; the UI still shows the outdated status
02WhenThe user refreshes the command list page
03ThenThe updated command status is shown in the list
04WhenThe user opens that command's detail page
05ThenThe detail view also reflects the updated status
Expected Result

After refresh, both the list row and the detail page show the most recent backend status. The outdated status is not cached or persisted anywhere in the UI.

Result: PASS FAIL BLOCKED N/A
@DMS-1006 · TC-UI-006 Failed and stale commands show correct UX treatment in list and detail US3-AC1
Preconditions
  • At least one command is in Failed state
  • At least one different command is in Stale state
Steps
#KeywordStep
01GivenA failed command and a stale command are both present in the command list
02WhenThe user views the failed command's row in the list
03ThenA failed indicator is shown; the row does not appear active or successful
04WhenThe user views the stale command's row in the list
05ThenA stale indicator is shown; the row does not appear active or successful
06WhenThe user opens the failed command's detail page
07ThenThe detail page shows the correct failed UX treatment (indicator, styling, no active-state affordances)
08WhenThe user opens the stale command's detail page
09ThenThe detail page shows the correct stale UX treatment (indicator, styling, no active-state affordances)
Expected Result

Failed and stale indicators are visually distinct from each other and from active or successful states. Treatments are consistent between list and detail views. Neither state exposes affordances that imply the command is still actionable.

Result: PASS FAIL BLOCKED N/A
